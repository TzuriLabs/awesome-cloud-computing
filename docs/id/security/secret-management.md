# Manajemen Rahasia

> Layanan dan alat untuk menyimpan, mengelola, dan mengakses informasi sensitif seperti kunci API, kata sandi, dan sertifikat di lingkungan cloud secara aman.

| Nama | Deskripsi | Tautan |
|------|-----------|--------|
| **AWS Secrets Manager** | Menyimpan dan mengelola rahasia seperti kunci API, kredensial database, dan token secara aman. | [AWS Secrets Manager](https://aws.amazon.com/secrets-manager) |
| **AWS Systems Manager Parameter Store** | Alternatif untuk Secrets Manager, mendukung parameter string aman. | [Parameter Store](https://docs.aws.amazon.com/systems-manager/latest/userguide/systems-manager-parameter-store.html) |
| **Azure Key Vault** | Menyimpan rahasia, kunci, dan sertifikat secara aman untuk digunakan oleh aplikasi dan layanan cloud. | [Azure Key Vault](https://learn.microsoft.com/en-us/azure/key-vault/general/overview) |
| **Google Secret Manager** | Menyimpan kunci API, kata sandi, sertifikat, dan data sensitif lainnya secara aman. | [Google Secret Manager](https://cloud.google.com/security/products/secret-manager) |

## Fundamental Manajemen Rahasia

### Jenis Rahasia

- **Kunci API** - Token autentikasi untuk layanan
- **Kredensial database** - Kombinasi nama pengguna dan kata sandi
- **Sertifikat** - Sertifikat SSL/TLS dan kunci privat
- **Kunci enkripsi** - Kunci untuk enkripsi/dekripsi data
- **Token OAuth** - Token autentikasi dan otorisasi
- **Kunci SSH** - Kredensial akses secure shell
