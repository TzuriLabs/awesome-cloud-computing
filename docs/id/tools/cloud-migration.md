# Alat Migrasi Cloud

> Alat dan layanan yang dirancang untuk mendukung penilaian, migrasi,
> dan transisi aplikasi, data, dan beban kerja ke lingkungan cloud
> atau antar penyedia cloud.

| Nama | Deskripsi | Tautan |
|------|-----------|--------|
| **AWS Migration Hub** | Layanan terpusat untuk melacak dan mengelola migrasi aplikasi menggunakan alat AWS dan mitra. | [AWS Migration Hub](https://aws.amazon.com/transform/) |
| **Azure Migrate** | Platform untuk menemukan, menilai, memigrasikan, dan mengoptimalkan beban kerja on-premises ke Azure. | [Azure Migrate](https://azure.microsoft.com/en-us/products/azure-migrate) |
| **Google Cloud Migration Center** | Antarmuka terpadu untuk menilai dan memigrasikan beban kerja ke Google Cloud. | [Migration Center](https://docs.cloud.google.com/migration-center/docs/) |
| **Carbonite Migrate** | Alat replikasi real-time untuk memigrasikan beban kerja fisik, virtual, dan cloud. | [Carbonite Migrate](https://cybersecurity.opentext.com/products/continuity-compliance/migrate) |
| **VMware HCX** | Platform migrasi perusahaan untuk memindahkan beban kerja VMware ke cloud publik dan privat. | [VMware Cloud Foundation Operations HCX](https://www.vmware.com/products/cloud-infrastructure/vcf-operations-hcx) |

## Strategi Migrasi

> Pendekatan yang umum dirujuk untuk memigrasikan beban kerja ke cloud,
> berbeda dalam upaya yang diperlukan dan tingkat perubahan arsitektur.

| Strategi | Karakteristik Kunci | Manfaat Tipe | Upaya |
|---------|---------------------|--------------|-------|
| **Lift and Shift (Rehosting)** | Pindahkan beban kerja dengan perubahan kode minimal atau tanpa perubahan | Migrasi lebih cepat, kompleksitas awal lebih rendah | Rendah |
| **Replatforming** | Buat penyesuaian terbatas untuk menggunakan layanan cloud terkelola | Performa lebih baik dan perbaikan operasional | Sedang |
| **Refactoring / Re-architecting** | Rancang ulang aplikasi menggunakan pola cloud-native | Skalabilitas yang ditingkatkan dan optimasi jangka panjang | Tinggi |
