# Manajemen Multi-cloud

> Alat dan platform untuk mengelola sumber daya di beberapa penyedia cloud,
> dengan fokus pada operasi terpadu, visibilitas, dan tata kelola.

| Nama | Deskripsi | Dokumentasi Resmi |
|------|-----------|-------------------|
| **Terraform Cloud** | Layanan terkelola untuk Terraform yang menyediakan fitur kolaborasi, manajemen state jarak jauh, dan penegakan kebijakan. | [Terraform Cloud](https://cloud.hashicorp.com/products/terraform) |
| **Pulumi Cloud** | Layanan terkelola untuk Pulumi yang menawarkan kolaborasi tim, kebijakan sebagai kode, dan fitur manajemen deployment. | [Pulumi Cloud](https://www.pulumi.com/product/pulumi-cloud) |
| **Crossplane** | Control plane berbasis Kubernetes open-source untuk mengelola infrastruktur dari beberapa penyedia menggunakan API deklaratif. | [Crossplane](https://www.crossplane.io) |
| **Scalr** | Platform manajemen Infrastructure as Code yang berfokus pada tata kelola, penegakan kebijakan, dan kontrol biaya. | [Scalr](https://scalr.com) |
| **env0** | Platform otomasi Infrastructure as Code berorientasi GitOps yang mendukung Terraform, Terragrunt, dan alur kerja terkait. | [env0](https://www.env0.com) |

## Mengapa Multi-cloud?

> Pendekatan multi-cloud sering diadopsi untuk memenuhi persyaratan teknis,
> operasional, atau regulasi tertentu. Poin-poin berikut merangkum
> pertimbangan yang umum dikutip.

### Potensi Manfaat

- **Kemandirian Vendor** - Mengurangi ketergantungan pada satu penyedia cloud
- **Pertimbangan Biaya** - Membandingkan harga dan opsi layanan di seluruh penyedia
- **Distribusi Risiko** - Menghindari ketergantungan satu penyedia untuk beban kerja kritis
- **Pemilihan Layanan** - Menggunakan layanan tertentu di mana paling sesuai dengan kebutuhan
- **Kebutuhan Kepatuhan** - Mengatasi residensi data atau batasan regulasi

### Tantangan Umum

- **Kompleksitas Operasional** - Mengelola beberapa platform dan API
- **Kebutuhan Keahlian** - Mempertahankan keahlian di seluruh penyedia
- **Konsistensi Keamanan** - Menerapkan kebijakan keamanan yang seragam
- **Visibilitas Biaya** - Melacak dan mengalokasikan pengeluaran di seluruh lingkungan
- **Upaya Integrasi** - Menghubungkan layanan dan alur kerja di seluruh cloud
